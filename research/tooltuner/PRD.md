---

## 0. 一句话

tooltuner 是一个**持续、开放式**的工具调用质量优化引擎,由 Claude Code agent 操作。

**核心信条:测量只约束「能 verify 什么」,绝不约束「能 imagine 什么」。** AI 想得宽——改描述 / schema / 系统 prompt / 教学 / 示例,甚至建议**拆 / 并 / 删 / 加工具**、**发明新的质量维度**、**质疑场景与指标本身**;paired-lift 只当**诚实闸门**(防自欺、防退化)。**没有终点,越转越好。**

---

## 1. 为什么 + 三条脊椎

- R1-R4 已证明:**模型能力不是瓶颈,契约设计才是**;但那是一次性手动大工程,焊死在 Forgify 91 工具。想要**低摩擦、随时随地、持续**优化。
- 三条血换的教训 = 本设计的脊椎:
  1. **分数不可靠**(±15 抖 + 测量假象)→ 别贪心修最低分。
  2. **优化无止境** → 没有"做完"。
  3. **别让流程把思路逼窄** → 测量是闸门**不是牢笼**;最值钱的改进常是**设计级**的(我们自己的 case→when 就是),不是描述微调。

---

## 2. 每一次"推" = 两个分开的动作(全篇的纲)

- **① 想得宽(divergent imagine)** — 从一切信号找改进点:不只低分,还有好工具的更优框架、raw 里的新失败、把赢搬给兄弟工具、没试过的杠杆、覆盖盲区;**以及元层面**——这工具**设计**对吗?该测的**维度**齐吗?**场景**够多样吗?
- **② 验得稳(honest verify)** — 先**读 raw 验信号真假**(防假象);能 A/B 的 → paired-lift + 跨工具回归,只留真赢;不能 A/B 的(设计级)→ 作为**给你的建议**浮上来。

> **信条:imagine 阶段无边界,verify 阶段才上闸门。别让闸门的形状反过来笼住 imagine。**

---

## 3. 它优化什么(范围:宽)

- **全部 LLM-facing 面**:工具描述 / schema / 系统 prompt / 教学守则 / 示例 / 分组。
- **质量维度可生长**:起步两轴 **SELECTION(选对)+ USAGE(用对)**;但 AI 发现新维度(效率/轮数、鲁棒性、**校准**=该不该用/何时反问、多工具编排、token 成本…)就**立新轴**。**两轴是种子,不是全集。**
- **工具集本身可被质疑**:产出包含"该拆/并/删/加、schema 重设计"这类**设计建议**(交给你,不自动改)。

## 4. 谁用 & 怎么用

- 操作者 = Claude Code agent(你的 session / `/loop`)。**不做调度 / 云端。**
- 想优化就开 session 让 AI「读 `research/tooltuner/`,跑一轮」。记忆持久,下次 / 换机器接着转。

---

## 5. 不是什么

- ❌ 不是人敲的 CLI / 框架 —— agent 当司机,PLAYBOOK 指路不绑手。
- ❌ 不是"修最低分 → 收敛做完" —— 开放式,无终点。
- ❌ 分数不是目标 —— **真实质量才是目标,分数只是 proxy**;paired-lift 是闸门不是靶。
- ❌ 不把**工具集 / 指标 / 场景**当不可变 —— 三者都能被质疑、被改、被扩。
- ❌ 不做 skill / standalone / GUI —— Claude Code 原生积木(Workflow / memory / `/loop`)+ 一个 markdown `PLAYBOOK`。
- ❌ **不当生产真相** —— 这是闭环,绝对分是 Claude 审美;**只信相对 / paired**,code 信**真执行**,真实使用日志待产品上线再补(详 SPEC §9 信任边界)。

## 6. 四层

| 层 | 是什么 |
|---|---|
| **① 记忆**(target 文件夹) | **三层(转 N 轮不糊)**:**当前真相** `STATE`/`CONCLUSIONS`/`ROUNDS`(大小不随轮数涨)+ **交付与活状态** `surfaces`(port 回 Forgify)/`axes`(可加维度)/`backlog`/`recommendations`/`changelog`/`scores` + **过程** `rounds/NNNN/`(不可变胶囊,可 GC)。详 SPEC §2 |
| **② 零件**(确定性) | `run_model` / `score`(多轴)/ `ab`(paired-lift + 跨工具回归)+ `gen` / `judge` Workflow。**= 现有代码泛化** |
| **③ PLAYBOOK.md**(方法论,皇冠) | 两动作 + 质量底线 + **反窄机制(§7)**;AI 跑一轮前读它 |
| **④ 司机 = agent** | 想得宽 + 验得稳,靠 ①②③ 撑 |

## 7. 反窄机制(本次重写的核心,钉进 PLAYBOOK)

1. **轴可生长** —— 发现新失败维度就立新轴,别只盯两轴。
2. **结构 / 设计提案是一等产出** —— 能说"该拆 / 删 / 加 / 重设计 schema",不只改文本。
3. **explore vs exploit** —— 规定一部分推是**发散**的:从零重写、狂想假设、best-of-N 不同框架打锦标赛,不只 vs 当前版微调 —— 破局部最优。
4. **场景 critic** —— 定期问"我们没测到啥情况",防优化到窄分布。
5. **反 Goodhart + outside view** —— 分数是 proxy;定期质疑"指标 / 场景漏了什么真实质量";引外部知识(别的工具生态怎么描述、新 prompting 研究、别的模型行为)。
6. **元认知殿后** —— 每轮收尾问一句:"我这轮是不是在局部爬山?有没有更大的、我没敢想的改法?"

## 9. 成功标准

- 再入 = 让 AI 读 `research/tooltuner/` 跑一轮,**零设置**。
- catalog 沿**可信指标** paired 累积变好(不退化)。
- **不止描述变好** —— 还稳定产出**设计级洞察 + 新质量维度**(= 没退化成窄微调器,这是本版第一标准)。
