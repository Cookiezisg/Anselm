# Foryx — Design（重做工作区）

> `design/` 是 Foryx 桌面端**重做的家**(干净重写,不在 `demo/` 上打补丁)。
> `demo/` = 当前形态参照(旧名 Forgify,维持不动);`design/` = 下一代、规范先行、把标准焊进代码。**新代码一律叫 Foryx。**

## 这里是什么

`demo/` 验证了**大骨架**(海洋 + 三岛 + 六契约 + Intent/Live + 组件库),这套保留。
但它**缺最关键的一层**:统一的「布局语法」——行高/间距/字阶/页骨架/交互全靠各海洋手搓,所以纷繁、会漂。
`design/` 先补这层,且用「文档解释 + token 定值 + 原语强制」三件套落地,让"照规范做"比"自己写 CSS"更省事。

## 怎么读

1. **先读 [`SPEC.md`](SPEC.md)** —— 全部规范(宪法)。这是你现在要研究的东西。
2. 你拍板后,再按 SPEC §8 的路线落 token → 原语 → 重写一个样板面 → 铺开。

## 一句话哲学

> **一致性是约束的副产品,不是精雕的结果。规范的力量在于它"拒绝"什么,不在于它"能表达"什么。**
