# 前端全量 i18n Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把前端 ~900 条 hardcode 中文 UI 文案全量改造成 react-i18next 驱动的中英双语,由 `settings.lang` 切换。

**Architecture:** react-i18next + 结构化 key(`t("account.noAccount")`),按 namespace 拆 `locales/{zh,en}/<ns>.json`,`import.meta.glob` eager 聚合。`settings.lang`(已存在)驱动 `i18n.changeLanguage`。分 9 批迁移,每批一个 commit + push。

**Tech Stack:** React 18, Vite 6, zustand, vitest, i18next + react-i18next(本计划新增)。

> 配套 spec:`docs/superpowers/specs/2026-05-26-frontend-i18n-design.md`(读它了解决策与边界)。

---

## 迁移准则(每批都遵守)

1. **只迁渲染层 UI 文案** —— 出现在 JSX 文本、`placeholder`/`title`/`aria-label`、传给 toast/`pushToast`、默认标题等**会显示给用户**的中文字符串。
2. **不动中文注释** —— 项目规范(§S11)要求中文注释,`//` 与 `/* */` 里的中文**保留原样**。逻辑层文件(`api/` `sse/` `store/` `hooks/` `bridge/` `motion/`)的中文大多是注释;只迁其中确实渲染/toast 的文案。
3. **不动非文案常量** —— 品牌色、provider abbr(`LLM_HINTS` 等)、枚举值不是文案。
4. **判断法**:这条中文会不会出现在用户屏幕上?会 → 迁;只在注释/常量里 → 不动。

## 单模块迁移 SOP(批 2–8 每个文件按此做)

以 `SettingsModal.jsx` 的账号区为**样板**(真实代码):

**Before:**
```jsx
<div className="set-acct-sub">本地工作空间 · 共 {users.length} 个</div>
{active?.displayName || active?.username || "无账号"}
<button className="set-pill-btn" onClick={...}>切换 / 新建</button>
<input className="set-acct-add-input" placeholder="新工作空间" ... />
```

**After:**
```jsx
import { useTranslation } from "react-i18next";
// ...组件内顶部:
const { t } = useTranslation("settings");
// ...
<div className="set-acct-sub">{t("account.localWorkspaceCount", { count: users.length })}</div>
{active?.displayName || active?.username || t("account.noAccount")}
<button className="set-pill-btn" onClick={...}>{t("account.switchOrNew")}</button>
<input className="set-acct-add-input" placeholder={t("account.newWorkspace")} ... />
```

**`locales/zh/settings.json`:**
```json
{ "account": {
  "localWorkspaceCount": "本地工作空间 · 共 {{count}} 个",
  "noAccount": "无账号",
  "switchOrNew": "切换 / 新建",
  "newWorkspace": "新工作空间"
} }
```

**`locales/en/settings.json`:**
```json
{ "account": {
  "localWorkspaceCount": "Local workspace · {{count}} total",
  "noAccount": "No account",
  "switchOrNew": "Switch / New",
  "newWorkspace": "New workspace"
} }
```

**SOP 步骤(单文件):**
1. 通读文件,圈出所有「渲染层中文」(按上面准则);注释/常量跳过。
2. 为每条想结构化 key(camelCase,语义分组,如 `account.noAccount`);动态值改插值 `{{var}}`;夹 JSX 的文案改 `<Trans>`(见下)。
3. 把 key 写进该 ns 的 `zh/<ns>.json`(中文原文)和 `en/<ns>.json`(英文翻译,风格对齐 onboarding en:简洁、技术、句末标点克制)。
4. 文件顶部 `import { useTranslation } from "react-i18next";`,组件内 `const { t } = useTranslation("<ns>");`,把中文替换成 `t("...")`。跨模块通用文案用 `t("common:save")`。
5. 跑该模块测试 `npx vitest run <相关 test 文件>`,绿。
6. `git add` 改动文件 + json,`git commit`(中文 commit,无 AI attribution),`git push origin main`。

**含 JSX 的文案用 `<Trans>`**(少数):
```jsx
import { Trans } from "react-i18next";
<Trans i18nKey="footer" ns="settings">数据本地存储于 <b>{{path}}</b></Trans>
// en: "Data stored locally at <b>{{path}}</b>"
```

- [ ] **Step 2: 写 `i18n/resources.js`(自动聚合)**

```js
// resources.js — eager-glob 所有 locales JSON,组装成 i18next resources。
// 加新 namespace 文件后这里零改动。

const mods = import.meta.glob("./locales/**/*.json", { eager: true });

export const resources = {};
for (const [path, mod] of Object.entries(mods)) {
  const m = path.match(/\.\/locales\/([^/]+)\/([^/]+)\.json$/);
  if (!m) continue;
  const [, lng, ns] = m;
  (resources[lng] ||= {})[ns] = mod.default;
}
```

- [ ] **Step 3: 写 `i18n/index.js`**

```js
import i18n from "i18next";
import { initReactI18next } from "react-i18next";

i18n.use(initReactI18next).init({
  resources,
  lng: useSettings.getState().lang,
  fallbackLng: "zh",
  defaultNS: "common",
  interpolation: { escapeValue: false },
  returnNull: false,
});

export default i18n;
```
