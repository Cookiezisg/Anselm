# WRK-042 — 生产级桌面 app「平台地基」调研总账

> **一句话**:把"一个真正生产级 Flutter 桌面 app 在视觉之下还要做的全部底层设计"摸清。这是 [[demo-roadmap]] Phase 4 动工**之前**用户要求加的一轮地基级调研——产出是这张**总账**,供逐项拍板「该不该做 / 何时做」。本篇为建造前的研究存档,落地后结论提取进 `concepts/` + `references/frontend/`(架构 / 平台地基章)并填 `landed-into`。

## 0. 执行摘要

- **现状判断**:`frontend-rebuild` 当前只有**视觉地基**(设计系统 + UI kit + 三岛骨架 + 窗口/缩放/i18n)。本总账盘的是**水面之下那 80%**——OS 集成、进程托管、发行签名、产品级运行时——目前**几乎全空白**(只落了窗口尺寸/居中 + 启动闪烁修复一小片)。
- **规模**:两轮调研合并 **41 个集群、217 项**(优先级:**P0 = 71** · P1 = 89 · P2 = 40 · P3 = 17),全部按原则 #8 给出**成熟包/机制**(而非手搓),并扎根我们的真实架构(Go sidecar / local-first 单用户 / durable scheduler / 三平台)。
- **两个战略地层**(本总账据此分册):
  - **分册 A — OS / 平台集成地基**(Pass 1,25 集群):窗口状态、托盘/生命周期、单实例、**sidecar 进程托管**、后台执行/能耗、开机自启、系统通知、深链/文件、全局热键/剪贴板、安全存储/权限、自动更新、**打包/签名/公证**、崩溃/日志/诊断、数据目录/迁移/备份、OS 主题/语言/无障碍、应用生命周期/状态恢复、原生菜单栏、文本输入(IME)、关于/许可、首次运行、启动预算、**loopback 安全**。
  - **分册 B — 产品级运行时 / 桌面交互**(Pass 2,16 集群,⭐ = 主调研**完全漏掉**):**流式取消 + block 树地基** ⭐ · **工具危险审批 + 动作披露** ⭐ · **Token/成本计量** ⭐ · **设置存储 + schema 版本迁移** ⭐ · **网络代理/TLS/离线降级** ⭐ · **durable 引擎可观测** ⭐ · 模型/Key 管理 · 命令面板/keymap · sidecar 打包+构建矩阵 · 文件选择器 · 指针/触控板/右键 · 错误边界壳 · 性能预算 · 顶层错误呈现。
- **最大收获**:跑第二个正交兜底网**确实值**——它从 8 个不同信息源枚举 227 项、折叠 50 项进 Pass 1(佐证 Pass 1 覆盖扎实),并捞出主调研整类漏掉的**产品/agentic 运行时地基**(上面 ⭐ 那批)。这些恰恰是 features 依赖的客户端核心管道。

## 1. 方法(可复核)

- **Pass 1**(workflow `wiaw0g3a8`,54 agent / ~238 万 token / 45 分钟):我定 16 类自上而下深研 → 完整性批判补漏(挖出 14 个缺口、补研 10 个) → 逐集群**对抗验证**(查包真实性/维护度/跨平台 + 优先级/相关性纠偏)。1 个 verify agent 失败(非关键)。

## 2. 大图景:三条主线

1. **跨切骨干(features 全都压在上面,必须 Phase 4 最先铺)**:`sidecar 进程托管`(抢端口/健康门控/优雅关停/**孤儿防护**)+ `loopback 安全`(绑 127.0.0.1 + **每次启动随机 bearer token** + Host 校验,防本机其它进程/网页打我们的本地服务)+ `数据目录/迁移` + `顶层错误边界壳` + `错误码→人话`。**没有这层,任何 feature 都没数据管道、也不安全。**
2. **OS 集成 polish(决定"像不像一个真 app")**:窗口记忆、关闭=退出 vs 托盘后台、系统通知(scheduler 跑完/待审批)、原生菜单栏、文本 IME 正确性、首次运行/空态。**多为中等成本、高可感**。
3. **发行与信任(临近 release 才阻塞,但决定能不能给别人用)**:macOS 签名+公证(**非沙盒 Developer ID**,因沙盒与"下载并执行运行时"冲突)、Windows Authenticode、自动更新(+ 同步更新 Go sidecar)、跨平台 sidecar 打包矩阵。

## 4. 对抗验证揪出的关键纠正(动手前务必采纳)

- **`window_manager.setBadgeLabel` 不能做 Dock/任务栏角标**(验证称这是该集群"最大事实错误")——用 `app_badge_plus`(macOS/部分)或原生通道;Windows 用 `windows_taskbar`(仅 Win)。
- **`flutter_markdown` 已 DISCONTINUED**(Google 标弃)——若要渲 release notes/markdown,改用维护中的替代(如 `flutter_markdown_plus`/`gpt_markdown`)或自渲。
- **`auto_updater`(Sparkle/WinSparkle)无 Linux** ——Linux 自动更新须另走(AppImage+zsync / 发行版仓库);或评估 `desktop_updater`(三平台,较新)。
- **`open_file_handler` 实质未维护、无 Windows/Linux** ——文件关联用平台原生注册(Info.plist / Windows 注册表 / `.desktop`),别押它。
- **多个包"陈旧但可用"**:`hotkey_manager` / `local_notifier` / `super_clipboard` / `launch_at_startup` / `windows_taskbar`——能用但要盯维护;通知首选 `flutter_local_notifications`(更活、三平台)。
- **`flutter` 的 highContrast 在桌面是 iOS-only**;**`RestorationMixin` 桌面不支持**(状态恢复须手做);**MS App Store / MAS 拒审**那套相关性多处不适用——我们走**非 MAS 的 Developer ID / Inno / AppImage** 直分发。
- 已剔除一个占位垃圾集群(`test`,agent 产出残渣)。
- 各集群表内 `⚠` 标在仅单平台/受限的包上;逐集群 verify 的"过度设计?"提示已折进表下。

## 5. 决策(2026-06-26 用户已拍板)

> 原 8 个开放问题已定如下;发行细节深挖见 [`WRK-043 发行 Playbook`](release-distribution-playbook.md)。

1. ✅ **关闭 = 后台续跑**(缩到托盘 + 后台,非退出)→ scheduler 后台模型成立;需托盘 + 关窗确认(有在途 run 时)。
2. ✅ **provider Key 归 Go 后端**(后端已持有)→ Flutter 端永不碰明文 Key,只调后端。
3. ✅ **macOS = 非沙盒 Developer ID + Hardened Runtime + 公证**;用户确认会买 $99/yr Apple Developer。不上 Mac App Store。
4. ✅ **loopback 三把锁进 P0**:绑 127.0.0.1 + 每次启动随机 `ANSELM_AUTH_TOKEN` + Host 校验(需后端小改)。
5. ◑ **要自动更新**(三平台);**appcast/下载托管**首选 GitHub Releases(抗封,旧 `anselm.host` 已被封)——最终落点待新域名/托管拍板(见 WRK-043 §5)。
6. ✅ **设置存储分阶段**:简单开关续用 `shared_preferences`;做 Settings feature 时再引入带版本迁移的 JSON 设置存(provider 设计成底层可替换)。
7. ✅ **v1 = 可发行给外部用户**(非仅 dogfood)→ 发行签名那批 P0 必做;详尽发行手册见 WRK-043。
8. ◑ **#8 处置**:#1 无障碍认证 / #3 in-app 帮助 = Pass 1 已够;**#4 商店元数据 + #5 法务许可已并入 WRK-043**;**#2 i18n 运行时格式化 + 时区/DST(尤其 Scheduler cron/夏令时)** = 真缺口,待晚点单独一小轮。

## 6. 落地建议(Phase 4 排序)

1. **先铺跨切骨干**(§2.1):随"运行时管道港进 rebuild"一起做——sidecar 托管 + loopback 安全 + 数据目录 + 错误边界 + 错误码呈现。**这批不做,Entities 等 feature 无法接真后端。**
2. **随 features 落地的产品运行时**(分册 B ⭐):流式取消/block 树、工具危险审批、Token 成本、durable 可观测——这些跟 Chat / Scheduler feature 绑定,在对应海洋建造时一并做。
3. **OS polish**(§2.2)穿插在 feature 之间补。
4. **发行/签名/更新**(§2.3)临近 release 集中做,但**架构决策(§5.3/§5.4)现在定**。
