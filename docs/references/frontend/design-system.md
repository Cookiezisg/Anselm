# 前端设计系统 + UI 套件（An*）

> demo 的视觉语言（`demo/core/tokens.css` + `core/primitives/*`）以 app 级严谨度移植到 Flutter:
> **设计令牌**（值源）+ **An\* 组件套件**（原语），让 features 纯组装、零 bespoke 样式。
> 本篇随套件 **逐组同提交** 填充（见 §4 路线）。架构/分层见 [`architecture.md`](architecture.md)；决策见 [`ADR 0004`](../../decisions/0004-frontend-flutter-architecture.md)。

## 2. 命名 + 纪律

- 组件类 `An<Name>`(`core/ui/`);文件 `an_<name>.dart`。纯框架无关模型在 `core/model/`(无 Flutter import)。
- **颜色/度量只走 token**:widget 内禁裸 hex/rgb/`Color(0x…)`/px(只有 `core/design` 可声明色值)——`make verify` 加 grep 门禁机械兜底（套件落地时接）。
- **悬停/选中底色淡入铁律**:`AnimatedContainer` 静止底用**目标色 alpha=0**——统一走 `Color.whenActive(active)` 扩展(`core/design/colors.dart`),绝不用 `Color(0x00000000)`(transparent black,lerp 到浅色经暗中点=暗闪)。
- **共享判定/令牌(去重单源)**:`Set<WidgetState>.isActive`(hover/press/focus 任一,`an_interactive.dart`)各控件统一取「视觉激活」;`AnOpacity.disabled` 统一禁用变暗;占满宽统一名 `block`(AnButton/AnInput/AnDropdown/AnActionGroup)。
- **交互基座 = `FocusableActionDetector`**(`AnInteractive`,原则 #8):hover/focus 由平台高亮模式驱动(焦点环只在键盘聚焦显)、Enter/Space 走标准 `ActivateIntent`、禁用即不可聚焦、`Semantics(button)` 无障碍。下拉菜单据此 + `FocusScope`/`FocusTraversalGroup` + autofocus 选中行得键盘导航(方向键 + Enter)。
- **文案只走 i18n**:严禁硬编码中英文,走 slang `context.t.<key>`(见 §3)。
- **`CustomPainter` 只配画没有身份的像素**(WRK-069 S5 立法):凡需要**焦点 / 语义层级 / Tooltip / 测试可寻址**的图元,**必须是真 widget**——painter 画出来的东西**没有身份**:`semanticsBuilder` 产的节点**在焦点树里根本不存在**(`rendering/custom_paint.dart` 全文件不创建任何 `FocusNode`),故纯 painter 下方向键遍历**物理上做不到**;且它的语义只能**平铺一层**(框架源码原话:「As of now CustomPainter does not support multiple tree levels.」),嵌套结构无从表达。**唯一解=混合架构**:**惰性家具**(轴 / 基线 / now 线 / 连线 / 网格)走 painter 底层 + `IgnorePointer`(零语义、零命中,绝不抢子件的点击),**有身份的图元**(事件点 / 节点 / 格)是 widget。先例 `AnGraphCanvas` · `AnScheduleTrack`。**推论**:「为了性能把这些点画成 painter」= 拿**可访问性与可测试性**换帧率,除非该图元**本就不需要身份**,否则不成交易。
- **桌面 a11y 的真实边界(先量再写,别照搬移动端习惯)**(WRK-069 S5 立法;engine `42d3d75a56` 逐行核 + 本地语义树 dump 实测):桌面三端(macOS/Windows/Linux)共走 `shell/platform/embedder/embedder.h` 的 C ABI,**四样东西到不了读屏**——① **`SemanticsRole`**:table/row/cell/columnHeader 自 3.32 起存在于 Dart,但 `embedder.h` **无 role 字段**(`grep -ci role` = 0),桌面 bridge 的角色**只由 flags 推导**(button/textField/header/image/link/radio/checkbox/switch/slider,兜底 group/staticText)→ **桌面表达不出任何表格结构**(且 `rowHeader` 连枚举都没有);② **`indexInParent` / 行列下标**:Dart · C++ · ABI **三层皆无**,`updateNode` 没有这个参数,没有绕道;③ **`liveRegion`**:`embedder.h` 有 `is_live_region` 字段但**桌面三端无人读它**(`ax_event_generator` 找不到 `kLiveStatus` 就什么都不 fire)→ **静默 no-op**(flutter#167318 至今 OPEN,**3.44.6 亦未修**——这条是**永久**边界,别指望升级捎带解决:与下面 (e) 那个**版本钉住、升级即消**的缺陷分属两类);④ **`hint`**:桌面 bridge 只拿它判「是否忽略此节点」,**内容永不外传**(AXHelp / UIA HelpText 恒空)。**桌面真正到得了的只有**:`label` / `value` / `tooltip`(仅 mac·win)/ `identifier` / `selected` / `focused` / custom actions / rect + 遍历序。**五条推论**:(a) **坐标、层级、"第几行共几行"这类结构信息必须编进 `label` 字符串**——没有第二个地方放;(b) **播报走 `AnA11y` 这条唯一缝**(底层 `SemanticsService.sendAnnouncement`——`announce` 已废,废因是**多窗口**、**不是**桌面坏;三桌面 embedder 都真消费它:mac→`NSAccessibilityAnnouncementRequested` / win→`NotifyWinEvent(kAlert)` / linux→ATK≥2.46),**绝不用 `liveRegion` 兜桌面**;**调用点绝不手写 `SemanticsService`**——平台规则(`announce` 三端推 vs `announceFocusMove` 只 mac)在调用点手写=每次重判一遍(WRK-069 S6 收编 6 处,`lib/` 现零手写);(c) **别拿 `MediaQuery.supportsAnnounceOf` 当能力探测**——那个位是**反向**的、且只有 Android 会设,它的含义是「本平台不劝阻 announce」而非「announce 出得了声」;(d) **「聚焦即被念」在 macOS 不成立**——mac bridge 把 `FOCUS_CHANGED` 归进「在 Mac 上没有意义」的跳过组,而 Windows(`kFocus`)/ Linux(`ATK_STATE_FOCUSED`)会发 → 键盘光标类控件须**只在 macOS** 补一发 `sendAnnouncement`(Windows/Linux 补=**双读**,flutter#153020);(e) **`selected: false` 是陷阱、不是「未选中」的同义词**(WRK-069 S6 逐层实证并**限定了它的有效期**)——`accessibility_bridge.cc:445` 把 tristate 直接塞进 bool 形参(`AddBoolAttribute(kSelected, flags->is_selected)`)而 `embedder.h:271` 的 `kFlutterTristateFalse` = **2**(真值)→ 显式 false 会被念成**已选中**。**macOS + Windows 共用该桥、都中招;Linux 不受影响**(`fl_accessible_node.cc:108` 显式比 `== kFlutterTristateTrue`)。**这是一处孤立疏漏、不是模式**——同文件的 `is_expanded`/`is_toggled`/`is_focused`/`is_checked` 全是显式比较、全对(故 `expanded`/`toggled` 照常传 bool,别跟着一起躲)。**⚠️ 它是版本钉住的、有退出条件**:上游已修(flutter#184058 / PR #184223,commit `d04108d6bf40`,2026-03-27),但晚于 3.41 切枝(2026-01-08)→ **3.41.x 全系带病**,而 **3.44.6 已含修复**。故当下说「否」唯一安全的方式是**不说**(`null`),**唯一裁决点 = `AnA11y.selected()`**(全 app 只此一处判、含完整机理与退出条件);**mise 一旦升离 3.41.x,删掉它、把 bool 直接透传**——未选中的可选面本该说 `hasSelectedState` + `!isSelected`(ARIA tab 契约),那才是真话,只是在 3/2 的平台上今天发不出去。**另**:`hasSelectedState` 的框架契约是「**此件可被用户选中**」(dart:ui 原话:未设此旗标即表示该 widget 不可被选中)→ **没有选中概念的控件(按钮/chip/复选框/开关/披露行)根本不该传 `selected`**,这一条与引擎缺陷无关、升级后依然成立:`AnInteractive.selected` 是 `bool?`,**null=「此处没有选中这个概念」、绝非 false 的同义词**。**并且:同一面语义旗标只许一层标注**——两个 `Semantics` 设同一旗标是**不兼容**配置,框架会把它们拆成父+子,`label` 会落在**被聚焦节点的孩子**上而不是它自己身上(读屏因此可能读不到);要么交给 `AnInteractive` 的 prop,要么只在自己这层标。
- **a11y 主张必须用语义树 dump 验,不许靠读**(WRK-069 S5):`ExcludeSemantics` 包在盒子上不等于包住了整格——`AnInteractive` / `AnTooltip` 各自也标语义,只排里面仍会留下**每格一个无 label 的按钮节点**(实测:20×24 格阵 1054 → 排错位置 575 → 排对 95)。写完 `Semantics` 必 dump(`rootSemanticsNode.toStringDeep(childOrder: DebugSemanticsDumpOrder.traversalOrder)`)看**真出来的节点**,别看自己以为写了什么。
- **动效 + 生命周期纪律(动画件铁律,源自 G2 调研 [`WRK-037`](../../archive/g2-feedback-states/README.md)):** ① **每个纯动效都附"成品感"静态兜底**(WCAG 2.2.2/2.3.3)——`AnMotionPref` 门控,reduced 时**不启动**循环(`.repeat()` 默认不被 `disableAnimations` 缩短)并渲染**收尾完成的静止帧**(Skeleton 静态哑底/Typewriter 完整主句/Stepper 瞬切/StatusDot 实心点);② **急切初始化**——controller/ticker/timer 绝不放 `late final =` 字段初始化器(惰性首次读可在 teardown 触发→崩),在 `initState` 赋值;③ **永远 dispose + mounted 守卫**;自重排 Timer 持有于字段、dispose 取消;④ **离屏即静默**——循环视觉走 vsync 控制器(TickerMode 离屏自停),Timer 仅用于离散相位切换;⑤ **单 ticker 默认**(`SingleTickerProviderStateMixin`);⑥ **作用域 rebuild**——`AnimatedBuilder(animation,builder,child)` + 循环叶子裹 `RepaintBoundary`;⑦ **异步状态播报走 `AnA11y.announce`**(唯一缝,内含平台规则;**`liveRegion` 在桌面三端是静默 no-op**、见下「桌面 a11y 的真实边界」,故它**不是**本项目的播报机制)——**但「异步状态」不等于「都该喊」**:只推**凭空出现、用户无从发现**的新信息(toast/提示条/整页翻错=**本身即新闻**);**本来就在**的家具只在**变化时**推(AnStepper 推进而非到场);一次渲很多个的**形状**根本不推(AnSkeleton:一屏 5–7 声「加载中」比沉默更糟)。装饰动效 `ExcludeSemantics`;⑧ **状态不靠色单独**(图标+文字+朗读词,WCAG 1.4.1)。matrix 门禁加 **reduced 轴**(每 specimen 在 `disableAnimations` 下须 `pumpAndSettle` 收敛——忘门控的循环卡死=FAIL)。
- **reduced 门控选择(G3 起明确,源 [`WRK-038`](../../archive/g3-rows-cards/README.md) §7):动效分三类、门控不同。** ① **循环/装饰动效**(shimmer / breath / typewriter / 光标闪烁)走 `reducedOrAssistive`(屏读活跃时持续动效是噪声);② **功能性一次性 reveal**(AnRowDetail 高度揭示、AnRow icon↔chevron 互换+旋转)走 `reduced`(真时 `Duration.zero` 即时到位、状态一致);③ **功能性微反馈/微过渡**(focus 边 / hover 提墨 / press / AnDropdown caret 旋转 / 就地编辑 idle↔editing swap)本就一次性 settle、天然过 matrix reduced 轴,**可不显式门控**(若用 `Animated*` 则 reduced 时取 `Duration.zero`)。**铁律:绝不用 `reducedOrAssistive` 门控功能性反馈 / reveal**(会剥夺屏读用户的 hover/focus/展开线索)——`reducedOrAssistive` 只留给装饰循环。

### G3 行与卡 · 建造日志 [WRK-038](../../archive/g3-rows-cards/README.md)

**共享几何/编辑原语（先于消费件落地）**
